# Contributing to .jos

Thank you for your interest in contributing to the `.jos` specification!

## The Open Standard

The `.jos` format is an **open standard** under MIT License. We welcome:

- Specification improvements
- New runtime implementations
- Documentation enhancements
- Bug fixes and testing

## How to Contribute

### 1. File Issues

Use GitHub Issues for:
- Specification clarifications
- Bug reports
- Feature requests

### 2. Submit PRs

1. Fork the repository
2. Create a feature branch
3. Make your changes
4. Run tests: `npm test`
5. Submit a PR

### 3. Specification Changes

For changes to the `.jos` format itself:

1. Open an RFC issue first
2. Discuss with maintainers
3. Draft specification changes
4. Submit PR with updated SPECIFICATION.md

## Code of Conduct

Be respectful. Be constructive. Focus on the work.
